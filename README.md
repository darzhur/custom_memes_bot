# 📦 Meme Bot + Supabase

## Общая архитектура

Проект состоит из двух частей:

1. **Telegram-бот**

   * Получает фото
   * Загружает на Catbox
   * Генерирует подпись через ProxyAPI
   * Сохраняет результат в Supabase

2. **Supabase**

   * Хранит мем-контекст (`memepedia`)
   * Хранит сгенерированные мемы (`generated_memes`)

---

# 🧠 Логика работы

## Поток:

1. Пользователь отправляет фото в Telegram
2. Бот:

   * скачивает изображение
   * загружает его на Catbox
3. Из Supabase (`memepedia`) берётся контекст последних мемов
4. ProxyAPI генерирует 3 подписи
5. Каждая подпись сохраняется в `generated_memes`
6. Подписи отправляются пользователю

# 🤖 ProxyAPI

## Важно для работы с изображением

Картинка обязательно передаётся как объект:

```json
{
  "type": "image_url",
  "image_url": {
    "url": "https://..."
  }
}
```

Текстовый контекст:

```python
{
  "type": "text",
  "text": f"Придумай 3 саркастичные подписи:\n{mcp_context}"
}
```
